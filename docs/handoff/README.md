# Handoff Notes

Use this folder for durable session-to-session continuity.

## When To Add A Handoff

- A task spans multiple sessions
- Project direction or architecture changed
- A new experiment or subsystem was started
- There are unresolved risks, blockers, or assumptions
- A new agent should be able to continue work without reading old chat

## Naming

Use `YYYY-MM-DD-short-title.md`.

Also keep `docs/handoff/LATEST.md` updated so fresh agents have a stable pointer.

## Minimum Content

- Goal of the session
- What changed
- What remains open
- Files that matter
- Recommended next actions

## Rule

If a future agent would be confused without a summary, create a handoff note before ending the session.
